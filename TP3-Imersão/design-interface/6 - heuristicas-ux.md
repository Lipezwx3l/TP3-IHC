# Introdução

Para avaliar a qualidade da interface desenvolvida, foram aplicadas as Heurísticas de Usabilidade de Nielsen, que são princípios reconhecidos internacionalmente para analisar se uma interface é clara, intuitiva, acessível e eficiente.

A aplicação dessas heurísticas permite identificar se o usuário consegue compreender facilmente as telas, realizar ações com segurança, navegar sem esforço, receber feedback visual adequado e interagir com o sistema de forma natural, sem precisar aprender ou decorar comandos.

A seguir, cada tela do aplicativo — incluindo suas versões em modo escuro — é analisada com base nessas heurísticas, destacando quais delas estão presentes e como contribuem para a experiência do usuário.


# 1. Tela Inicial — Timer Pomodoro

![Imagem do WhatsApp de 2025-11-28 à(s) 22 53 26_ada5f5df](https://github.com/user-attachments/assets/b7c5dd27-2b84-4757-9ed5-79babaef209b)

Figura 1 - Timer Pomodoro

Heuristicas encontradas são:

| Heurística                                    | Como aparece na tela                                                                                           |
| --------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| **#1 – Visibilidade do estado do sistema**    | O tempo (25:00), o botão de iniciar e o contador de metas (1/10) mostram claramente o estado atual.            |
| **#2 – Compatibilidade com o mundo real**     | Uso de ícones universais (play, metas, menu, tema escuro) e o timer circular que representa passagem de tempo. |
| **#3– Consistência e padronização**          | Ícones, cores e layout iguais às outras telas; barra de navegação sempre na mesma posição.                     |
| **#4 – Prevenção de erros**                   | Botões bem espaçados, ícones claros e sem funções complexas que possam causar erro.                            |
| **#5 – Reconhecimento em vez de memorização** | O usuário sabe que 1/10 representa progresso de metas sem precisar lembrar. Ícones autoexplicativos.           |
| **#6 – Flexibilidade e eficiência de uso**    | Presença do modo escuro/claro e navegação rápida entre telas.                                                  |
| **#7 – Design estético e minimalista**        | Poucos elementos, foco no timer, sem excesso visual.                                                           |
| **#8 – Ajuda e documentação**                | Elementos intuitivos, mesmo sem texto explicativo.                                                             |

# 4. Versões em Modo Escuro

Figuras 4, 5, 6 - Telas no Modo Escuro

Heuristicas encontradas são:

| Heurística                                              | Como aparece no modo escuro                                                    |
| ------------------------------------------------------- | ------------------------------------------------------------------------------ |
| **#1 – Visibilidade do estado**                         | Contraste adequado permite leitura clara tanto no modo claro quanto no escuro. |
| **#2 – Consistência**                                   | As telas mantêm o mesmo layout, apenas mudam as cores.                         |
| **#3 – Prevenção de erros**                             | A troca de tema não altera botões, posições ou funções.                        |
| **#4 – Flexibilidade**                                  | Usuário escolhe quando alternar entre tema claro e escuro.                     |
| **#5 – Estética e minimalismo**                         | Visual limpo, sem poluição visual em modo escuro.                              |
| **#6 – Ajuda para usuários com necessidades especiais** | Tema escuro auxilia pessoas com sensibilidade à luz, dislexia ou fotofobia.    |

   
# Conclusão

A análise das telas com base nas heurísticas de Nielsen demonstrou que o protótipo atende aos principais princípios de usabilidade esperados para uma interface acessível e funcional.

Mesmo sem funcionalidades completas implementadas, as telas apresentam comunicação visual clara, consistência entre os elementos, navegação intuitiva, linguagem simples, uso correto de metáforas visuais e feedback coerente, especialmente na contagem de metas (1/10), troca de telas e alternância entre modo claro e escuro.

As heurísticas mais presentes foram:

✔ Visibilidade do Estado do Sistema,

✔ Consistência e Padronização,

✔ Reconhecimento em vez de Memorização e

✔ Estética e Design Minimalista,

o que indica que o usuário consegue entender as funções das telas sem esforço, mesmo que ainda não sejam interativas.

Assim, o protótipo se mostra visualmente consistente, informativo e acessível, constituindo uma base sólida para as próximas etapas de implementação funcional e de validação com usuários.
